# ClarityLoop: Uncertainty Is Decorative — Deterministic Authority-Boundary Release Control Is What Makes Agent Workflows Safe to Ship

**Working paper, v1 — 2026-06-18.** Reproducible artifacts: `research/experiments/`, prediction ledger `research/results.tsv`, deterministic benchmark `packages/evals`.

## Abstract

Self-improving agent *harnesses* now raise task success substantially (HarnessX: +14.5% avg), but optimize capability with essentially no governance: a capability-tuned agent in our benchmark commits an unsafe action in 36% of risk-bearing cases. We ask a separable question — not *can the agent produce the artifact?* but *should this artifact commit now?* — and study a deterministic **release-control layer** that maps a structured operational state to one of `commit / escalate / reject / promote`. Our headline result is a **negative** one, and it is the contribution we are most confident in: the layer's named *uncertainty/entropy* mechanism is **decorative** on realistic agent-release load. Ablating it changes false-commit, safe-completion, and completion by **0.00 pp** on a 36-case benchmark, and the outcome is **invariant** as the entropy threshold sweeps its entire range. The measured safety — 0% false-commit at 86% completion, versus 36% / 100% for a capability-only agent — comes entirely from a **deterministic authority-boundary gate plus an evidence-gathering loop**, not from uncertainty estimation. We then ask, honestly, whether uncertainty is ever load-bearing, and find it is — but only in a *diffuse* regime where no single hard violation exists, and only when the threshold is *calibrated*: on a held-out diffuse corpus with materiality labels defined independently of the entropy weights, a calibrated weighted threshold reaches 98.3% vs 78.3% for a naive signal count and 28.3% for ablation. Finally, because the agent emits the very state the gate reads, we characterize the **trust boundary** by adversarial emission: corrupting model self-report defeats only 1 of 6 unsafe archetypes, whereas compromising the independent verifier layer defeats 4 of 6 — so the guarantee reduces to extraction integrity, and the design rule is *re-derive safety signals, never trust self-report*. Capability and governance are separable; the governance that matters is discrete authorization, not continuous uncertainty.

## 1. Introduction

The agent literature of 2024–2026 splits cleanly. One branch makes agents *more capable* by evolving the harness — prompts, tools, memory, control flow — from execution traces (HarnessX [1], Agentic Harness Engineering [2], ADAS [3], AFlow [4], Gödel Agent [5]). These report consistent task-success gains and treat safety, almost without exception, as out of scope. A second branch measures how *unsafe* agents are: indirect prompt injection succeeds 24% of the time against ReAct-GPT-4 (InjecAgent [6]), 84% on average across Agent Security Bench [7], and frontier models violate safety in >54% of stateful-workspace action sequences (Saber [8]). The gap between the two branches is governance: a principled layer that decides *when an agent-authored action is clear, safe, and authorized enough to commit*.

The project's original framing claimed that **uncertainty estimation** drives this decision — an "uncertainty-aware release control" layer. This paper is the result of taking that claim seriously enough to try to kill it. It does not survive. Our central finding is that on realistic release load the uncertainty mechanism does no work at all, and that the safety wins come from a much less glamorous place: a deterministic gate that checks an **authority boundary** (is this action category permitted without sign-off?) and a **risk class** (is the action reversible / below the auto-commit ceiling?), fed by an **evidence loop** that resolves recoverable gaps before deciding. We report the negative result first because it is the most robust thing we know, and because the honest version of the contribution is stronger than the promotional one: *capability and governance are separable, and the separable governance layer is discrete and verifiable, not probabilistic.*

We make four empirical claims, each tied to a falsifiable experiment and a pre-registered prediction (the prediction ledger is an artifact):

1. **Decorative uncertainty (robust).** Ablating the commit-entropy branch changes nothing (0.00 pp) on the 36-case benchmark, invariant to the threshold (§5.1).
2. **Where the wins live.** A capability-only agent commits 36% unsafe; the same artifacts behind the authority-boundary gate + evidence loop commit 0% unsafe at 86% completion; a gate without the loop collapses to 31% completion (§5.1).
3. **Uncertainty rehabilitated, but scoped and calibrated.** In a diffuse regime with no hard violation, a *calibrated* weighted-uncertainty threshold generalizes on held-out materiality labels (98.3% vs 78.3% naive count), while an uncalibrated threshold leaks a third of cases (§5.2).
4. **The trust boundary.** The 0% guarantee is conditional on emission integrity; corrupting self-report defeats 1/6 archetypes, corrupting the verifier layer 4/6 (§5.3).

## 2. Related work

**Harness evolution (the capability axis we are not).** HarnessX [1] and peers optimize task success from traces; none gates release on safety. This is our wedge — *harnesses evolve; release control decides what ships* — and our benchmark includes a HarnessX-like baseline (`harness_evolution`: resolves evidence well, no gate) as the sharpest comparison.

**Uncertainty / calibration (why entropy is the wrong primitive alone).** Semantic-entropy methods [9] and successors estimate generation uncertainty, but Semantic Energy [10] is built on entropy's *failure* in identifiable regimes; *From Entropy to Calibrated Uncertainty* [11] shows raw entropy is uncalibrated until Platt-scaled; and the one method that gates *actions* on entropy-style uncertainty, *When to ASK* [12], obtains zero in-domain benefit and warns against "simple combination." Our §5.1–§5.2 reproduce this critique from inside a deployed gate: uncalibrated thresholds are inert or leaky; calibration is the lever. Crucially, *no prior work validates calibrated uncertainty as a gate for irreversible agent actions* — the white space §5.2 enters.

**Authorization gates / runtime governance (where our wins live).** A 2025–2026 cluster builds pre-execution authorization: AgentSpec [13] (DSL runtime enforcement, 90%+ unsafe-exec prevented), Progent [14] (least-privilege per tool call), MAC for agents [15], and *Before the Tool Call* [16] (deterministic pre-action authorization). The single prior work claiming *uncertainty-aware* release control is ReDAct [17] — token-level uncertainty deferring to a larger model, single-axis and model-to-model, with no authority/risk/human term. No framework unifies authorization, risk, and (calibrated) uncertainty in one commit decision; that conjunction is our method (§3), with the honest caveat that the uncertainty term is a residual (§5.1).

**Injection defenses / evaluation (the external axis).** AgentDojo [18] is the standard dual-axis (utility + attack-success) testbed; the strongest reported defense, PromptArmor [19], sanitizes *inputs* to <1% ASR but cannot help once detection misses. An action-time authorization gate is complementary: it blocks the unauthorized action even when the payload is never detected. Release-management work (Automated Self-Testing [20], LLM Readiness Harness [21]) gates CI promotion on absolute readiness thresholds; none gates on a *safety delta vs the incumbent* (§3.4).

## 3. The ClarityLoop release-control layer

**3.1 Structured state, deterministic decision.** The agent (any harness) emits only a schema-validated `LatentWorkflowState` — known facts, missing fields (required/optional), claims (with/without evidence pointers), risk flags, policy flags, stale-memory refs, tool failures. The model never produces a score or a decision. Deterministic code computes everything downstream. This is what makes capability and governance separable: the decision function reads the structured state + policy, never the model's weights, so the same gate sits in front of any agent.

**3.2 The commit predicate.** The gate is a priority-ordered conjunction; the first matching branch decides:

> commit ⇔ (no hard-reject verifier fails) ∧ (no unresolved *required* field) ∧ (risk class ≤ auto-commit ceiling) ∧ (action category ∉ approval-required set) ∧ (no unmitigated high-severity hazard) ∧ (evidence coverage ≥ floor) ∧ (residual commit-entropy U < τ)

The first six terms are **hard, discrete, and independently verifiable**. The seventh, U, is the only continuous term. We do **not** claim these arise from a single optimization objective — that would be a unification we have not demonstrated. We claim the weaker, true thing: U is a *residual* that can only bind when every hard term has already passed (proven decorative in §5.1).

**3.3 The evidence loop.** Before deciding, ClarityLoop runs verifiers and gathers evidence for recoverable gaps (missing fields, unsupported claims). This is the only difference between ClarityLoop and a "fixed gate": the fixed gate bounces uncertain cases to a human (31% completion); the loop resolves the recoverable ones first (86%). Both share the identical commit predicate — so the completion gain is attributable to the loop, not the gate's strictness.

**3.4 Promotion as safety-weighted regression gating.** Promotion (workflow vN→vN+1) lifts the same predicate to a replay set: promote ⇔ candidate's safe-completion ≥ incumbent's ∧ false-commit ≤ incumbent's on recorded traces. This is champion-challenger / shadow evaluation with a *safety-dominance* acceptance criterion; we present it as sound engineering, not a novel mechanism.

## 4. Experimental setup

**ClarityLoopBench.** 36 cases across 12 archetypes (clear, ambiguous, stale-memory, supplier/catalog mismatch, missing-delivery, unauthorized-discount, unsupported-claim, adversarial-attachment, policy-exception, high-value). Each carries objective ground truth (safe-to-raw-commit, missing-resolvable, requires-approval, adversarial, policy-violation-if-auto-commit, resolution-reliable, hard-gap). **One uniform scorer** computes safety/false-commit/ASR identically for all five baselines — this contract is frozen and is what makes the comparison honest (an earlier rigged version with per-baseline hardcoded outcomes was discarded; see §6). A **deterministic provider** (no LLM) drives the bench so it measures the decision logic, not model variance — a limitation we return to in §6.

**Baselines.** `bare_qwen` (no governance), `dynamic_qwen` (naive gap-filling), `harness_evolution` (HarnessX-like: resolves evidence, no gate), `fixed_gate` (commit predicate, no loop), `clarityloop` (loop + predicate). The strong baseline is `harness_evolution` — the capability-SOTA analog — not a strawman.

**Validation experiments** (separate corpora, not the frozen bench): the diffuse-regime study (§5.2, `entropy_regime.ts`, `entropy_calibration.ts`) and adversarial emission (§5.3, `adversarial_emission.ts`). All use the shipped `runCommitGate` / `scoreEntropy`, not reimplementations.

## 5. Results

**5.1 Uncertainty is decorative; authorization is decisive (Fig 1, Fig 2).**

| baseline | completion | false-commit |
|---|---|---|
| bare_qwen | 100% | 92% |
| dynamic_qwen | 100% | 56% |
| harness_evolution | 100% | 36% |
| fixed_gate | 31% | 0% |
| **clarityloop** | **86%** | **0%** |

Against the capability-only `harness_evolution`, ClarityLoop converts 36 pp of unsafe commits into zero at a cost of 14 pp completion (the "constraint tax"). Disabling the entropy branch (threshold 0.3→1.0) changes ClarityLoop's false-commit, safe-completion, and completion by **0.00 pp**; sweeping τ ∈ {0.10, 0.20, 0.30, 0.50, 0.90} leaves the 86%/0% headline **identical** (Fig 2). Mechanistic explanation: whenever uncertainty is high, a hard branch (reject / needs-more-info / risk / authority) has already decided; and no single entropy signal reaches τ, so entropy could only bind on co-occurring soft signals the bench never isolates. The prediction (0 pp, recorded before running) was confirmed. **Uncertainty does no work here; the gate + loop do all of it.**

**5.2 Rehabilitating uncertainty: the diffuse regime, calibrated, held-out (Fig 4).** We construct the regime entropy *should* own: states with no hard violation (no required-missing field, risk ≤ L1, no high-severity flag, coverage above floor) but accumulated soft signals. Here entropy is the only branch that can fire. With it ablated, the gate commits **100%** of should-escalate cases; the uncertainty term is genuinely load-bearing in this regime. But at the shipped τ=0.3 it leaks **33%** (cases at 0.275/0.287 fall just below) — a calibration failure reproducing [11] from our own gate. To test generalization rather than fit, we built a 120-case diffuse corpus labelled by a **materiality rule independent of the entropy weights**, tuned τ on a calibration half, and scored a held-out half: calibrated weighted entropy **98.3%** vs naive soft-signal count **78.3%** vs ablation **28.3%**. The weighting generalizes because it encodes decision-materiality (an unsupported claim outweighs a stale-memory ref), a distinction a count cannot make — and it is honestly imperfect (≈1.7% boundary-collision error). **Uncertainty is real but scoped: it earns its keep only in the diffuse region, and only when calibrated.**

**5.3 The trust boundary (Fig 3).** The gate reads state the agent emits, so we corrupt each layer over 6 unsafe archetypes. Honest: **0%** false-commit. Corrupting model self-report (`emit-corrupt`): **17%** — only `fraud_risk` falls, the one archetype defended solely by a model-emitted flag. Corrupting the independent verifier/extraction layer (`extract-corrupt`): **67%** — the four verifier-defended archetypes fall. Both: **100%**. The guarantee therefore reduces to **extraction integrity**: ClarityLoop shrinks the trusted surface from "the whole agent" to "the deterministic verifier layer," and the design rule is explicit — *re-derive every safety-relevant signal; never trust self-report* (the lone self-report-only defense is the weak link).

## 6. Discussion — including what went wrong

**The journey, honestly.** The project began as "uncertainty-aware autopilot." An adversarial self-review found the first benchmark was **rigged** — per-baseline hardcoded outcomes, a strawman fixed gate, an entropy threshold that never bound. We rebuilt it with one uniform scorer (the §4 contract) and the headline survived but the *attribution* flipped: the wins were the gate, not the entropy. A pre-registered ablation then confirmed entropy was decorative (0.00 pp). Rather than bury that, we made it the headline, and asked where — if anywhere — uncertainty is real (§5.2). A theory review flagged three weaknesses we then addressed empirically: circularity (fixed by the held-out calibration split), the emission trust assumption (characterized in §5.3), and a "unification" overclaim (dropped; U is a residual, §3.2). The negative result made the paper stronger, not weaker.

**Limitations.** (1) The bench is deterministic — it isolates the decision logic but assumes faithful structured emission; §5.3 bounds how that assumption fails but a live-LLM study is future work. (2) The materiality labels in §5.2, though independent of the entropy weights, are author-defined; a different labeling could favor the naive count. (3) The external AgentDojo evaluation (gate as injection defense; baseline to beat: PromptArmor <1% ASR [19]) is harness-complete but requires a live key and is user-gated; we report it as external validation, not orchestrator-measured. (4) ClarityLoop reduces but does not eliminate the trusted surface — a compromised extractor defeats it, as it would any gate.

**Takeaway.** The marketable story (uncertainty decides what ships) is false on realistic load. The true story is more useful: governance is separable from capability, and the governance that works is *discrete authorization re-derived independently of the agent*, with calibrated uncertainty a scoped residual for diffuse doubt. Build the gate; calibrate the uncertainty; trust the extractor, not the agent.

## 7. Conclusion

We set out to validate uncertainty-aware release control and instead falsified its central claim, then rebuilt a defensible one. On a uniform-scored benchmark, deterministic authority-boundary gating plus an evidence loop eliminates unsafe commits (36%→0%) at modest completion cost (14 pp), entirely without the uncertainty mechanism the approach was named for. Uncertainty is real only in a calibrated, diffuse, scoped regime, and the whole guarantee rests on re-deriving safety signals rather than trusting agent self-report. Harnesses evolve; release control decides what ships — and what decides is the gate.
